# 全容器操作流程(宿主机只当显示器)

> 目标:**所有指令与代码都在容器里跑,宿主机只跑 X server**(显示 rviz/rqt 窗口)。宿主机不装、不跑任何 ROS / UE 代码。
>
> 本机是 Intel+NVIDIA 混合显卡(Optimus)。容器内 Vulkan **无法把 UE 渲染窗口 present 到宿主桌面**(swapchain 阶段 `VkResult=-13` 崩),所以 UE 走**离屏渲染**(不开窗口),可视化用 **rviz(雷达点云+位姿)+ rqt_image_view(相机画面)**,这两个**软件渲染**从容器 X11 转发到宿主,不抢独显。
>
> 注:键盘遥控(drone_teleop)已从本仓库移除。无人机控制由**你的算法节点**(发布 `vel_body_cmd`)给出;调试时也可用 `rosservice call .../takeoff` 起飞。

---

## 一、架构总览

```
宿主机(只跑 X server / DISPLAY=:0)
   │  xhost +local:  放行容器访问 X
   │
   ├─ 容器1  simulator01      roscore + UE(离屏)        --net host --gpus device=0   无窗口
   ├─ 容器2  simulator01-viz  rviz / rqt_image_view     --net host + X11(显示窗口)    窗口在宿主
   └─ 复用 sim 容器           rostopic/rosservice CLI   docker exec sim01 ...        无窗口
```

所有容器 `--net host` → 共享宿主网络栈 → 都连 `localhost:11311`(容器1 里 roscore 起的)。容器1 的 `start.bash` 入口自己起 roscore,**不用单独起 roscore**。CLI 直接 `docker exec sim01` 复用模拟器容器(它有 ROS Noetic),不必另起容器。

## 二、镜像(已构建好;重建见 `Dockerfile` / `Dockerfile.viz`)

| 镜像 | 基于 | 内容 |
|---|---|---|
| `simulator01` | `Dockerfile` | UE 二进制 + ROS Noetic(基础)。`ADD` 本地 `Build/` + 烤 `settings.json`。入口 `start.bash`(起 roscore + UE) |
| `simulator01-viz` | `Dockerfile.viz`(`FROM simulator01`) | rviz / rqt / mesa |

## 三、操作流程(裸 docker 命令,逐项说明)

> 命令默认在 `~/RMUA/simulator/` 下执行(`cd ~/RMUA/simulator`)。

### Step 1 ｜ 放行容器访问宿主 X(每个登录会话做一次)

```bash
xhost +local:
```
允许本机进程(含容器)连宿主 X server。rviz 要靠它**显示窗口**。

### Step 2 ｜ 启动模拟器 = roscore + UE 离屏(终端 1,前台)

```bash
cd ~/RMUA/simulator
docker run -it --rm --name sim01 --net host \
  --cpuset-cpus="0-9" --gpus 'device=0' \
  -e Seed=123 \
  simulator01
```
- `-it`:前台交互,看 UE 日志,`Ctrl+C` 停。
- `--rm`:退出即删容器;`--name sim01`:便于 `docker exec`/`logs`/`stop`。
- `--net host`:用宿主网络栈 → 容器里 roscore 起在宿主 `localhost:11311`,别的容器/CLI 能连。
- `--cpuset-cpus="0-9"`:钉 0–9 核,留核给 viz;`--gpus 'device=0'`:GPU 0 = NVIDIA(离屏渲染靠它,不 present 故 Optimus 不崩)。
- `-e Seed=123`:随机种子,选赛道。入口 `start.bash` 读 `$Seed` → `RMUA.sh -RenderOffscreen seed 123`。

等终端不再刷日志、UE 起来后,进行下一步。

### Step 3 ｜ 验证 roscore+UE 起来了(终端 2)

复用 sim 容器跑 ROS CLI(它有 Noetic):
```bash
docker exec sim01 bash -lc 'source /opt/ros/noetic/setup.bash && rostopic list | grep drone_1'
```
能看到 `/airsim_node/drone_1/imu/imu` 等话题即正常。
> 嫌长可用 `./scripts/ros.sh rostopic list`(就是这条命令的封装,自动 source Noetic + devel)。

### Step 4 ｜ 可视化(终端 3)

rviz(雷达点云 + 位姿 + 相机面板,软件渲染不抢独显):
```bash
cd ~/RMUA/simulator
docker run -d --rm --net host --name viz_rviz \
  -e DISPLAY=$DISPLAY -e ROS_MASTER_URI=http://localhost:11311 \
  -e LIBGL_ALWAYS_SOFTWARE=1 -e QT_X11_NO_MITSHM=1 \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  -v ~/RMUA/simulator/rmua_full.rviz:/tmp/rmua_full.rviz \
  --entrypoint bash simulator01-viz -c \
  'source /opt/ros/noetic/setup.bash && exec rviz -d /tmp/rmua_full.rviz'
```
- `-d`:后台跑,rviz 窗口在宿主桌面弹出;`docker rm -f viz_rviz` 关闭。
- `LIBGL_ALWAYS_SOFTWARE=1`:**关键**,软件渲染,避免和 UE 抢独显。

rqt_image_view(看四路相机原图):
```bash
docker run -it --rm --net host \
  -e DISPLAY=$DISPLAY -e ROS_MASTER_URI=http://localhost:11311 \
  -e LIBGL_ALWAYS_SOFTWARE=1 \
  -v /tmp/.X11-unix:/tmp/.X11-unix \
  --entrypoint bash simulator01-viz -lc \
  'source /opt/ros/noetic/setup.bash && rqt_image_view'
```

### Step 5 ｜ 控制无人机(起飞 / 速度指令)

调试起飞(等价调 takeoff 服务):
```bash
./scripts/ros.sh rosservice call /airsim_node/drone_1/takeoff "{waitOnLastTask: true}"
```
正式飞行靠**你的算法节点**发布 `/airsim_node/drone_1/vel_body_cmd`(`airsim_ros/VelCmd`),在算法容器里跑(参考 `../IntelligentUAVChampionshipBase/basic_dev/src/`)。重置当前局:
```bash
./scripts/ros.sh rosservice call /airsim_node/reset
```

### Step 6 ｜ 查状态(任意终端)

```bash
./scripts/ros.sh rostopic echo -n1 /airsim_node/drone_1/debug/pose_gt   # 位姿真值
./scripts/ros.sh rostopic hz   /airsim_node/drone_1/imu/imu              # IMU 频率(应≈100)
./scripts/ros.sh rostopic echo -n1 /airsim_node/drone_1/vel_body_cmd    # 当前速度指令
```

### Step 7 ｜ 收尾

- viz:`docker rm -f viz_rviz`(或 `Ctrl+C`)。
- 模拟器:终端 1 `Ctrl+C` 停 UE+roscore。
- 关 X 放行(可选):`xhost -local:`

## 四、关键注意

1. **ROS 命令在 Noetic 容器里跑**:宿主是 ROS Humble,直接 `rosservice` 会 `ImportError: cannot import name 'Log'`。用 `docker exec sim01` 或 `./scripts/ros.sh`。
2. **`vel_body_cmd` 的 `header.stamp` 必须是仿真时钟(IMU stamp)**,墙上时钟会被当过期丢弃——算法节点里要订阅 `imu/imu` 取 stamp 盖戳。
3. **VEL 与 PWM 不可同一场混用**:规则手册"控制接口类型以首次接收的信号类型为准",首发锁定,另一种被静默忽略。
4. **指令频率上限 100Hz**(规则 §2.3)。
5. 工厂内 **GPS=0**,巡检靠视觉/雷达定位。
